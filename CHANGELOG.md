## v1.4.0

### 🆕 新功能

- **自动更新**：启动检测 GitHub Releases，弹窗展示更新内容，一键下载并安装
- **外部文件关联**：支持从文件管理器/社交软件直接打开 .nmbak 备份文件
- **定稿上限**：设置中自定义最大定稿章节数，超限批量淘汰减少缓存重建
- **智能排序**：文件树和定稿列表按章节号数值排序，第10章不再排在第2章前
- **AI 定稿查询**：AI 可通过 `listFinalized` 工具查看所有定稿章节
- **项目删除完善**：删除项目时同步清理外部存储目录

### 🔧 优化

- **聊天加载性能**：首次加载历史记录直接定位，消除长列表动画卡顿
- **共享定稿实例**：编辑器与 AI 使用同一 FinalizedManager，定稿状态实时同步

### 🐛 修复

- 修复 AI 无法感知用户手动标记的定稿章节
- 修复更新检测版本号比较逻辑（本地 versionName vs GitHub tag）
- 修复单项目备份 ZIP 结构不一致导致无法恢复

## v1.2.1

### 🆕 新增

- **Plan / Agent 双模式**：AI 辅助写作分阶段进行，Plan 模式专注大纲规划，Agent 模式负责正文创作
- **对话栏内联选项**：AI 提问时直接在聊天气泡内显示选项按钮，支持自定义填写
- **Token 用量统计**：设置中查看各项目 Token 消耗（系统提示词 / 对话 / 工具调用），带可视化进度条
- **定稿章节标记**：文件树长按标记定稿，已定稿章节纳入缓存前缀以降低 Token 消耗
- **系统提示词管理**：AI 自动维护，Plan 阶段收集的设定自动导入，切换模式时导出到项目目录

### 🔧 优化

- 消息构建顺序遵循 DeepSeek KV-Cache 最佳实践，大幅提升缓存命中率
- 聊天记录管理显示项目名称，删除项目时自动清理关联数据
- 模式状态持久化，退出编辑器重进不丢失

### 🐛 修复

- 修复启动图标资源缺失导致构建失败
- 修复部分矢量图标文件损坏（零字节）
- 修复聊天记录中不完整工具调用链导致的 API 400 错误
- 修复选择框同时出现时点击无响应
- 修复 `choice` 角色消息误传给 API 导致的解析错误
